# 麻雀点数計算トレーナー 開発ロードマップ

最終更新: Ver2.10.5

## 開発方針

- 小さく実装する
- 十分に動作確認する
- Gitへコミットする
- Versionごとに完成条件を定義し、一区切りつけてから次へ進む

設計改善は機能追加に合わせて段階的に実施する。

## Version 3

Version 3では、点数計算トレーニングとしての使いやすさを維持しながら、鳴きや槓を含む実戦再現性を高める。

### 優先度A: 副露対応

#### 対象

- チー
- ポン
- 大明槓
- 加槓
- 暗槓

#### 実装内容

- 副露面子の表示
- 手牌と副露面子を区別したレイアウト
- 門前・非門前の判定
- 副露による翻数変化
- 門前限定役の不成立判定
- 食い下がり役の翻数変更
- 副露を含む問題データ形式の追加
- 鳴きあり問題の追加
- 自動採点および解説の副露対応

### 優先度A: 槓ドラ・槓裏ドラ表示

槓子数に応じて、表ドラ表示牌と裏ドラ表示牌の枚数を増やす。

#### 表示枚数

- 槓なし: 表ドラ1枚、裏ドラ1枚
- 槓1回: 表ドラ2枚、裏ドラ2枚
- 槓2回: 表ドラ3枚、裏ドラ3枚
- 槓3回: 表ドラ4枚、裏ドラ4枚
- 槓4回: 表ドラ5枚、裏ドラ5枚

裏ドラは、立直して和了した問題で表示する。

#### 実装内容

- 表ドラ表示牌の複数表示
- 裏ドラ表示牌の複数表示
- 問題データのドラ表示牌を配列で管理
- 槓子数とドラ表示枚数の整合性チェック
- 複数のドラ表示牌からドラ枚数を計算
- 監査ツールの複数ドラ対応
- 既存の暗槓問題の表示見直し

### 優先度A: 明槓対応

#### 対象ケース

- ポン済み面子への加槓
- 他家の捨て牌による大明槓
- 明槓を含む和了形
- 明槓を含む符計算

#### 確認事項

- 明刻・明槓・暗刻・暗槓の符の違い
- 中張牌と么九牌の符の違い
- 副露による役・翻数への影響
- 槓ドラ追加タイミングの表現方法

### 優先度B: 問題拡充

副露および槓を含む問題を追加する。

例:

- チーして成立する三色同順
- チーして食い下がる一気通貫
- ポンした役牌
- ポンした対々和
- 明槓を含む三槓子
- 暗槓と明槓が混在する問題
- 副露した混一色・清一色
- 槓ドラが複数乗る問題
- 立直後の暗槓と槓裏ドラを扱う問題

問題数の目安は、内容を優先し、必要に応じて150～200問程度まで拡充する。

### 優先度B: 苦手問題の優先出題

- 問題ごとの正答率保存
- 間違えた問題の優先出題
- 苦手な符・翻数の優先出題
- 役別・符計算要素別の出題
- 回答時間が長い問題の再出題

## 設計改善

機能追加に合わせて必要な範囲で段階的に実施する。

### 実施済み

- 問題データを `questions.js` へ分離

### 今後の候補

- `app.js` へのロジック分離
- `styles.css` へのスタイル分離
- 役判定ロジックのモジュール化
- 符計算ロジックのモジュール化
- 点数計算ロジックの共通化
- アプリ本体と監査ツールで計算ロジックを共有
- 問題データ形式のスキーマ化

設計改善のみを目的として大規模な作り直しは行わず、必要な機能追加と合わせて進める。

## テスト方針

1. 変更内容と確認範囲を整理する
2. 検証版を作成する
3. 動作確認する
4. 必要に応じて修正する
5. 本番版へ反映する
6. Gitへコミットする

本番問題へテスト専用問題を直接混在させない。

副露・槓対応では、以下を重点的に確認する。

- 門前と非門前の役判定
- 食い下がり
- 明刻・暗刻・明槓・暗槓の符
- 槓数とドラ表示枚数
- 複数ドラ表示時の翻数
- 既存問題への回帰影響
