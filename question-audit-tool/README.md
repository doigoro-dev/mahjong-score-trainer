# 問題データ監査ツール

## 概要

麻雀点数計算トレーナーで利用する `questions.js` を監査するための補助ツールです。

`app/questions.js` を `question-audit-tool/questions.js` にコピーして利用します。

## Version 0.9

監査項目は次の12項目です。

1. 問題ID重複チェック
2. 必須項目チェック
3. 翻数整合性チェック
4. 符整合性チェック
5. 点数カテゴリ整合性チェック
6. 切り上げ満貫整合性チェック
7. 役の翻数合計チェック
8. 役一覧整合性チェック
9. 点数カテゴリ計算チェック
10. 最終点数表示チェック
11. 牌コード・牌枚数チェック
12. ドラ・裏ドラ形式チェック

## ドラ・裏ドラ形式チェック

次の項目を検証します。

- `doraIndicators`
- `uraDoraIndicators`

### 配列形式

両項目が配列であることを確認します。

空配列も正常です。

```javascript
doraIndicators: []
uraDoraIndicators: []
```

### 有効な牌コード

数牌は次の形式です。

```text
1m ～ 9m
1p ～ 9p
1s ～ 9s
```

字牌は次の形式です。

```text
east
south
west
north
white
green
red
```

### 配列内の重複

同じ表示牌が同じ配列内に複数登録されている場合はエラーにします。

```javascript
doraIndicators: ["3m", "3m"]
```

表ドラと裏ドラの間で同じ牌が使われることはあり得るため、
異なる配列間の重複はエラーにしません。

## 将来のカン対応

`doraIndicators`と`uraDoraIndicators`は複数の表示牌を格納できるため、
カンによって表示牌が増える仕様にも対応できます。

Ver0.9では形式監査に限定し、次の項目は将来対応とします。

- カン回数とドラ表示牌枚数の整合性
- リーチ有無と裏ドラ表示牌枚数の整合性
- 表示牌から実際のドラを算出するチェック
- 手牌内のドラ枚数と翻数の整合性

これらは、将来 `kanCount` や副露情報を問題データへ追加した段階で
監査項目を拡張します。

## 牌コード・牌枚数チェック

次の項目の牌コードと枚数を検証します。

- `concealedTiles`
- `winningTile`
- `concealedKans`

通常の和了形は、次の合計が14枚になることを確認します。

```text
concealedTilesの枚数
+ winningTileの1枚
+ concealedKansの組数 × 3枚
```

同一牌の物理的な枚数は4枚以内である必要があります。

## 最終点数表示チェック

符・翻・親子・ツモ／ロン・点数カテゴリから点数を再計算し、
次の項目を検証します。

- `answer.score.pointText`
- `answer.score.display`

`display`は既存アプリの仕様に合わせ、次の両形式を正常とします。

```text
8000点
```

```text
満貫（8000点）
```

## 起動方法

1. `app/questions.js`を監査ツールのフォルダへコピーする
2. `index.html`をブラウザで開く
3. 「監査開始」を押す
