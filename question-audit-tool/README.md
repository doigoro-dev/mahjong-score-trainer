# 問題データ監査ツール Ver1.0

## 概要

麻雀点数計算トレーナーで使用する `questions.js` を、
ブラウザ上で監査するための補助ツールです。

問題追加やデータ修正後に実行し、ID・符・翻・点数・牌構成などの
基本的な不整合を早期に発見することを目的としています。

このツールは問題内容のすべてを保証するものではありません。
役の成立条件や待ち形など、機械的に監査していない内容については
アプリ上での表示確認と目視レビューも併用してください。

## フォルダ構成

```text
question-audit-tool/
├── index.html
├── styles.css
├── audit.js
├── questions.js
└── README.md
```

ZIPには `questions.js` を含めていません。
監査時は、アプリ本体の最新ファイルをコピーして使用します。

```text
app/questions.js
        ↓ コピー
question-audit-tool/questions.js
```

## 使用方法

1. `app/questions.js` を `question-audit-tool/questions.js` へコピーする
2. `question-audit-tool/index.html` をブラウザで開く
3. 読み込み件数を確認する
4. 「監査開始」を押す
5. 「異常」と「エラー件数」が0であることを確認する
6. エラーがある場合は、問題IDと監査項目を基に `questions.js` を修正する
7. 修正後、もう一度監査を実行する

## Ver1.0の監査項目

監査項目は次の12項目です。

1. **問題ID重複チェック**  
   同じ問題IDが複数登録されていないか確認します。

2. **必須項目チェック**  
   問題表示、正解、管理情報、ドラ情報に必要な項目が存在するか確認します。

3. **翻数整合性チェック**  
   `answer.totalHan` と `management.han` が一致するか確認します。

4. **符整合性チェック**  
   `answer.fu` と `management.fu` が一致するか確認します。

5. **点数カテゴリ整合性チェック**  
   `answer.score.category` と `management.scoreCategory` が一致するか確認します。

6. **切り上げ満貫整合性チェック**  
   `answer.score.kiriageMangan` と `management.kiriageMangan` が一致するか確認します。

7. **役の翻数合計チェック**  
   `answer.yaku[].han` の合計が `answer.totalHan` と一致するか確認します。

8. **役一覧整合性チェック**  
   `answer.yaku[].name` と `management.mainYaku` の内容が一致するか確認します。

9. **点数カテゴリ計算チェック**  
   符と翻から、通常・切り上げ満貫・満貫・跳満・倍満・三倍満・数え役満を再計算します。

10. **最終点数表示チェック**  
    符・翻・親子・ツモ／ロン・点数カテゴリから支払点を再計算し、
    `answer.score.pointText` と `answer.score.display` を確認します。

11. **牌コード・牌枚数チェック**  
    手牌・和了牌・暗槓の牌コード、和了形の構成枚数、同一牌の上限を確認します。

12. **ドラ・裏ドラ形式チェック**  
    表示牌配列の形式、有効な牌コード、同じ配列内の重複を確認します。

## 点数表示の扱い

`pointText` は、再計算した点数と完全一致する必要があります。

```text
8000点
2000点／4000点
4000点オール
```

`display` は既存アプリのデータ仕様に合わせ、次の両形式を正常とします。

```text
8000点
```

```text
満貫（8000点）
```

点数カテゴリ自体は、点数カテゴリ計算チェックで別途検証します。

## 牌コード

数牌は次の形式です。

```text
1m ～ 9m
1p ～ 9p
1s ～ 9s
```

字牌は次の形式です。

```text
east
south
west
north
white
green
red
```

## 暗槓を含む枚数計算

暗槓は物理的には4枚ですが、和了形の構成上は1面子なので、
構成枚数の計算では3枚として扱います。

```text
concealedTilesの枚数
+ winningTileの1枚
+ concealedKansの組数 × 3枚
= 14枚
```

同一牌の上限確認では、暗槓を実際の4枚として数えます。

## ドラ・裏ドラ

次の項目は配列として管理します。

```javascript
doraIndicators: []
uraDoraIndicators: []
```

複数の表示牌を格納できるため、将来のカンによるドラ追加にも対応可能です。

Ver1.0では形式監査に限定し、次の内容は将来拡張とします。

- カン回数とドラ表示牌枚数の整合性
- リーチ有無と裏ドラ表示牌枚数の整合性
- 表示牌から実際のドラを算出するチェック
- 手牌内のドラ枚数と翻数の整合性
- 明槓・加槓を含む副露情報の監査

これらは、将来 `kanCount` や副露情報を問題データへ追加した段階で対応します。

## 推奨運用

問題データを変更したときは、次の順番で確認します。

```text
questions.jsを修正
↓
監査ツールへコピー
↓
12項目の監査を実行
↓
アプリ本体で対象問題を表示
↓
問題文・牌姿・正解表示を目視確認
↓
Gitへコミット
```

監査結果がすべて正常でも、問題の役成立や待ち形が正しいことまでは
保証しないため、変更した問題のアプリ表示確認は省略しないでください。

## Ver1.0の位置づけ

Ver1.0を問題データ監査ツールの初期完成版とします。

今後は新機能を継続的に増やすのではなく、次の場合に必要な範囲だけ更新します。

- アプリ本体の問題データ構造を変更した場合
- 新しい牌表現や副露情報を追加した場合
- 既存監査に不具合が見つかった場合
- 実戦モードでカンやドラの管理情報が追加された場合
